Often in programming you'd have to make a decision on whether or not to reinvent the wheel. That is, whether to create something entirely from scratch or utilize existing libraries and APIs. Many of us grapple with a sense of guilt when leaning on others' code to swiftly develop a project. It can almost feel like taking a shortcut.

I often feel guilty when using libraries or existing api's when building something. I'm sure most of us deal with this feeling. It somehow feels like cheating to use other's code and quickly build the project.

Until now, I've been choosing to reinvent the wheel. Part of the reason for that behaviour is that I wanted to learn how things worked at the low level -- to go deep into concepts so that I break the abstractions and know exactly how stuff works.

> Although breaking abstractions is a good thing for learning, it is not suitable for building actual things out into the real world. The abstractions exist for a reason. We want to build on top of stuff rather than building different variations of the same stuff. Then only can we advance as a civilization.

I decided to not learn anymore stuff unless it's really needed in the immediate process of building something. I accept the fact that I can't know it all. No one can. It is better to add value on the top of stuff we already have rather than trying to reinvent the entire thing from scratch. It is okay to use existing libraries and tools to build something that will provide value to our civilization.

In the end, all that matters is that we build beautiful products that solve humanity's problems, one by one.
